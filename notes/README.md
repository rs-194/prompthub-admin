# notes 学习笔记说明

## 1. notes 目录用途

`notes/` 用于存放个人学习笔记、面试复习资料和模块理解记录。

它不是项目公开说明文档，和 `docs/` 的职责不同：`docs/` 更偏项目维护、公开展示和架构记录，`notes/` 更偏开发者自己的学习与复习。

## 2. 目录结构说明

```text
notes/
├─ README.md
└─ interview/
   ├─ prompt-module-notes.md
   └─ prompt-module-qa.md
```

- `prompt-module-notes.md`：提示词管理模块第一阶段代码理解笔记。
- `prompt-module-qa.md`：提示词管理模块第一阶段面试问答扩展版。

## 3. 使用方式

- 开发新模块后，可以在 `notes/interview/` 中补充对应模块的理解笔记。
- 面试前可以优先阅读对应模块的 notes。
- notes 可以比 docs 更口语化，但仍需保持清晰。

## 4. notes 与 docs 的区别

| 目录 | 面向对象 | 内容类型 | 是否偏公开 |
| --- | --- | --- | --- |
| docs | 项目维护者、GitHub 访客、面试官 | 项目说明、架构、路线、开发记录 | 偏公开 |
| notes | 开发者自己 | 学习笔记、面试复习、代码理解 | 偏个人 |

## 5. 更新规则

- 当某个模块新增较复杂逻辑时，应补充或更新对应 notes。
- 当面试回答、代码理解、模块数据流发生变化时，应同步更新 notes。
- notes 不需要记录每次小改动，但需要跟上模块核心逻辑变化。
- 不允许把未实现功能写成已完成。
- 如果功能仍是 mock 数据，需要明确标注。

## 6. 公开仓库说明

如果项目未来公开到 GitHub，可以根据需要选择是否上传 `notes/`。

如果 `notes/` 中包含个人复习、面试准备或较私人的学习记录，可以选择通过 `.gitignore` 忽略。

如果决定公开，应保持表达正式，避免出现过度私人化内容。
